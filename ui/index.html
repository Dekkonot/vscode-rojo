Build Roblox games using Rojo directly from Visual Studio Code and sync your files in to Roblox Studio. This extension will keep Rojo up-to-date and acts as a pass through to the managed Rojo binary seamlessly behind the scenes. You can start and stop Rojo with one click, while still harnessing all of the power that Rojo has to offer.

Please Support Development on Patreon if you've found this extension useful!

News

  • Rojo v0.5.x is now out of alpha! Check out this page for details on how to migrate your project.
  • You can use the "Rojo: Convert rojo.json to default.project.json" option in the Command Panel (Ctrl+Shift+P) to attempt to migrate automatically.
  • The "releaseType" option which previously allowed you to select pre-release or release has been removed in favor of a "releaseBranch" setting which allows you to specify the major release version you want to use. 0.5.x is now the default major release version.
  • You can still bind projects to specific Rojo versions by setting the "targetVersion" option at the workspace level.

Roblox Studio Plugin

You need to install a plugin into Roblox Studio to use Rojo! You can either install the plugin from the Roblox plugin page (but then you will need to click "Update" in the Studio plugins panel every time Rojo updates), or you can let this extension manage and install the plugin for you.

You must choose one of these two options in order to use Rojo with Roblox Studio.

Tips

  • Want to have more than one Rojo project open at once? VS Code allows you to add multiple root folders to your workspace, and this extension supports that out of the box! Right-click on a blank background portion of the Explorer panel, and click "Add folders to workspace". As long as a default.project.json file exists at the top level of that folder, you'll be able to use this extension with as many Rojo projects as you want to have open at once.
